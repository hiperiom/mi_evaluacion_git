Objetivo de la Actividad

En esta evaluación mediremos tus conocimientos sobre buenas prácticas simulando un flujo de trabajo real colaborativo usando Git.
Tus conocimientos serán evaluados en base a tu capacidad para clonar un repositorio, crear ramas, realizar commits atómicos con mensajes descriptivos, modificar archivos HTML siguiendo una plantilla y finalmente subir tu trabajo al repositorio remoto para su revisión.

1 Clonar y Ramificar

Primero, debes descargar en tu computadora el repositorio remoto (branch principal) que el profesor ha compartido:
https://github.com/hiperiom/mi_evaluacion_git.git
Para ello, abre tu terminal y ejecuta la clonación. Luego, accede a la carpeta del proyecto. Ya dentro de ella, crea una rama local (branch) exclusiva para tu trabajo para no afectar la rama principal (main) y cambiate a ella. El formato recomendado para nombrar tu rama es: feature/tu_nombre_apellido_cedula. Ejemplo: feature/paulo_joaquin_22014778. 

git clone [URL_DEL_REPOSITORIO_REMOTO]
cd [NOMBRE_DE_LA_CARPETA]
git checkout -b feature/tu_nombre_apellido_cedula

Nota: Sustituye los corchetes con los datos reales.

2 Crear la Página Local

Ya en tu rama creada, dirígete a la carpeta llamada paginas_estudiantes/ que ya viene en el proyecto. Crea una copia de esta plantilla y renómbrala con tu nombre, apellido y cédula siguiendo el formato: (ej. tu_nombre_apellido_cedula.html).

3 Flujo de Commits (Versionado)

A medida que vayas construyendo tu página, debes registrar tu progreso realizando 5 commits atómicos (commits pequeños y específicos). Aquí tienes el estándar que debes usar:

  • Commit 1: Crear tu archivo HTML base usando plantilla.
    git commit -m "feat: crear archivo HTML base a partir de plantilla"
  • Commit 2: Personalizar la plantilla con tu información.

    Edita en tu archivo los textos entre corchetes con tus datos reales:

    • [Título del Tema Asignado]
    • [Subtítulo o descripción breve del punto a investigar]
    • [Tu Nombre y Apellido]
    • [Tu Cédula]
    • [Tu Correo en classroom]

    Asegúrate de mantener la estructura HTML tal como fue diseñada y solo cambia lo que se te pide.

    git commit -m "feat: Personalizar archivo HTML con datos personales"
  • Commit 3: Añadir contenido sobre tu tema asignado.

    Edita en tu archivo los textos entre corchetes con el tema que te tocó exponer:

    • [Escribe aquí la introducción a tu investigación...]
    • [Punto importante 1...], [Punto importante N...]
    • [Conclusión o resumen final de tu investigación...]

    Asegúrate de mantener la estructura HTML tal como fue diseñada y solo cambia lo que se te pide.

    git commit -m "docs: agregar contenido sobre [tu tema asignado]"
  • Commit 4: Añadir botón de retorno.
    Reemplazar el comentartio html:
    <!-- AQUI DEBE IR UN BOTON DE RETORNO AL INDEX PRINCIPAL -->
    Por el siguiente código:
    <a href="../index.html" class="btn-back">Volver al Catálogo Principal</a>
    Asegúrate de mantener la estructura HTML tal como fue diseñada y solo cambia lo que se te pide.
    git commit -m "feat: añadir botón de retorno"

4 Integración en el Catálogo Principal

Ahora debes vincular tu página. Abre el archivo index.html (el que está en la raíz). Busca la fila de la tabla donde está tu nombre y cédula (Si no la encuentras, créala).

Notarás que en la columna de investigación dice Enlace a tu pagina. Debes borrar ese texto y reemplazarlo por una etiqueta <a> que apunte a tu archivo, colocando como texto visible el tema real que te cocó exponer en el video.

<!-- DEBE CAMBIAR DE ESTO: -->
<td>Enlace a tu pagina.</td>

<!-- A ESTO: -->
<td><a href="./paginas_estudiantes/tu_nombre_apellido_cedula.html">Funciones de la RAM</a></td>
  • Commit 5: Vincular página local en el index principal.
    git commit -m "feat: vincular página local en el index principal"

5 Subir al Remoto (Push)

Como tu rama feature/... fue creada en tu computadora y no existe aún en el servidor remoto, un git push normal arrojará error. Para subirla y rastrearla correctamente, debes usar la bandera -u de esta manera:

git push -u origin feature/tu_nombre_apellido_cedula

¡Listo! Al hacer esto, el docente podrá revisar tu rama, evaluar tus commits y fusionar tu trabajo a la rama main.

5 Sincronización Continua (Pull)

Tu trabajo no termina al hacer push al remoto.
Para mantener tu repositorio local actualizado con los aportes y validaciones que el profesor realice en la rama main, debes asegurarte de sincronizar frecuentemente.
Es el profesor quien realizará la fusión de tu rama a main cuando revise tu trabajo, asi como también el de tus compañeros.
Cada vez que el profesor haga merge de una rama a main, debes actualizar tu repositorio local para reflejar esos cambios y tener la versión más reciente. 

# Cambiate a la rama main
git checkout main

# Trae las actualizaciones del servidor
git pull origin main

# Si necesitas regresar a tu rama de trabajo para seguir editando
git checkout feature/tu_nombre_apellido_cedula

Nota: Esto garantiza que siempre estés trabajando sobre la versión más reciente del catálogo y que tu repositorio local no se desvíe del progreso colectivo de la clase. Siempre actualiza main y no tu rama de trabajo.
